먼저 아래 순서대로만 진행해라.

1) 가장 먼저 읽을 파일
- AGENTS.md
- project_context.md
- ROADMAP.md
- TASKS.md

2) project_context.md를 읽고 프로젝트 목적, 구조, 기술 스택, MVP 범위를 이해해라.

3) ROADMAP.md를 읽고 현재 개발 Phase와 이후 확장 흐름을 파악해라.

4) TASKS.md를 읽고 현재 Phase의 "다음 작업"만 확인해라.
- 현재는 Phase 1 (MVP 개발)이다.
- TASKS.md의 "다음 작업" 범위를 벗어나는 기능은 구현하지 마라.

5) 작업 시작 전 현재 프로젝트 폴더를 빠르게 스캔해서
- 주요 디렉토리
- 이미 존재하는 핵심 파일
- 새 파일을 넣어야 할 적절한 위치
를 짧게 요약해라.

6) 실제 코드 작성 전에
- 이번 작업 목표
- 수정/생성할 파일
- 구현 순서
를 5~10줄 정도로 짧게 설명해라.

7) 기존 구조를 불필요하게 전면 수정하지 마라.
- 큰 구조 변경이 필요하면 먼저 영향 범위를 설명하고 멈춰라.

8) 작업 완료 후 반드시
- TASKS.md 상태 업데이트
- CHANGELOG.md 기록
을 수행해라.

9) 실행 방법, 설치 방법, 사용 방법이 바뀌면 README.md를 갱신해라.

10) 구조, 범위, 입출력 정의가 바뀌면 project_context.md를 갱신해라.

11) 코드 작성만으로 완료 처리하지 마라.
- 가능한 경우 실행 또는 테스트로 확인해라.
- 실행/테스트를 못 하면 이유와 사용자가 직접 확인할 항목을 남겨라.

12) 필수 정보 부족, 외부 연동 불명확, 현재 작업 범위 초과, 테스트 실패 원인 미확정 상태에서는 억지로 확장 구현하지 말고 보고해라.
