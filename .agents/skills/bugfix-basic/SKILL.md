# 역할
오류를 최소 수정으로 해결한다.

# 언제 쓰나
- TypeScript 에러
- 빌드 실패
- 런타임 예외
- 상태 업데이트 오류

# 핵심 작업 지침
1. 오류 메시지를 먼저 그대로 확인한다.
2. 재현 경로를 가장 짧게 정리한다.
3. 최소 수정으로 해결 가능한지 먼저 판단한다.
4. 필요한 경우 로그를 추가한다.
5. 데이터 문제인지 UI 문제인지 상태 문제인지 먼저 분류한다.
6. 큰 리팩토링은 마지막 수단으로만 사용한다.
7. 수정 후 실행 또는 테스트로 재확인한다.
