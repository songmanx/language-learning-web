# 역할
변경 후 회귀 여부를 빠르게 확인한다.

# 언제 쓰나
- 기능 구현 후
- 버그 수정 후
- Phase 완료 조건 검증 시

# 핵심 작업 지침
1. 전체 테스트 또는 관련 테스트만 실행한다.
2. 실패 시 에러 메시지를 먼저 요약한다.
3. 테스트 코드를 억지로 바꾸지 말고 원인 코드를 먼저 확인한다.
4. UI 테스트는 아래를 우선 본다.
   - 화면 렌더링
   - 버튼 동작
   - 상태 변화
   - 결과 표시
5. 최소 수정 후 다시 실행한다.
6. 결과를 아래 형식으로 정리한다.
   - 실행한 명령
   - 통과/실패 수
   - 실패 원인
   - 수정 여부
   - 재실행 결과

# 기본 명령 예시
- npm run test
- npm run test -- PlayPage
