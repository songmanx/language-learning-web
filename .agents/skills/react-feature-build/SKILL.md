# 역할
현재 프로젝트 구조를 유지하면서 React 기능을 구현한다.

# 언제 쓰나
- 새 페이지 생성
- 새 컴포넌트 추가
- Zustand 스토어 작성
- 게임 로직/로더/UI 구현

# 핵심 작업 지침
1. 먼저 기존 파일 구조와 네이밍 규칙을 확인한다.
2. 기능은 가능한 한 아래 단위로 나눈다.
   - 페이지
   - 컴포넌트
   - 스토어
   - 서비스
   - 유틸
3. 한 번에 너무 많은 기능을 묶지 않는다.
4. 모바일 우선 UI를 기본으로 구현한다.
5. 외부 연동이 미정이면 mock 또는 placeholder를 먼저 둔다.
6. 상태와 UI를 분리하고, 재사용 가능한 컴포넌트는 분리한다.
7. 구현 후 테스트가 가능한 최소 단위까지 확인한다.

# 체크포인트
- 라우트 연결 여부
- Zustand 상태 분리 여부
- props / types 분리 여부
- 모바일 UI 사용성
- 기존 구조와 충돌 여부
