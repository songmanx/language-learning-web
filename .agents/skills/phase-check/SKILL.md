# 역할
현재 프로젝트가 어느 Phase인지 확인하고, 지금 해야 할 작업 범위를 제한한다.

# 언제 쓰나
- 작업 시작 전
- 새 기능 요청을 받았을 때
- 범위 초과 여부를 판단할 때

# 핵심 작업 지침
1. `ROADMAP.md`에서 현재 개발 Phase를 확인한다.
2. `TASKS.md`에서 현재 Phase 목표, 완료 조건, 다음 작업을 확인한다.
3. 현재 요청이 다음 작업 범위를 넘는지 판단한다.
4. 넘는 경우 구현하지 말고 아래 형식으로 보고한다.
   - 범위 초과 이유
   - 현재 Phase 우선 작업
   - 권장 다음 순서
5. 현재 작업이 적절하면 다음 작업 항목 번호를 명시하고 진행한다.

# 출력 예시
- 현재 Phase:
- 현재 목표:
- 이번 요청이 속하는 작업:
- 범위 초과 여부:
- 바로 진행할 작업:
