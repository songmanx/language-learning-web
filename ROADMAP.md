## Phase 1: MVP 개발

### 목표

일본어 1개 언어 기준으로 로그인, 언어 선택, 기본 플레이, 결과 화면, 최소 저장 구조까지 동작하는 MVP를 만든다.

### 포함 기능

- 프로젝트 초기 세팅
- 로그인 화면
- 언어 선택 화면
- 일본어 단어 데이터 로드
- 기본 플레이 화면
- 점수 / 하트 / 콤보
- 세션 종료 payload 생성
- `Review_State` 최소 반영
- 기본 테스트 환경

### 제외 기능

- 업적 시스템
- 고급 통계
- 이미지 문제
- 듣기 모드 고도화
- 자기채점 작문 완성형
- 다국어 추가
- 관리자 UI

### 완료 조건

- 로그인 동작 가능
- 언어 선택 후 플레이 진입 가능
- 기본 플레이 루프 동작
- 세션 종료 payload 생성 가능
- 최소 테스트 통과

### 목표

MVP 기능을 안정화하고 저장 실패, 캐시, fallback, 테스트 범위를 보강한다.

### 포함 기능

- API 에러 처리
- 저장 실패 대응
- 캐시 복구 보강
- 테스트 보강
- 모바일 UI 조정
- 로그 개선

### 제외 기능

- 다국어 추가
- 업적 전체 구현
- 고급 통계
- 관리자 UI

### 완료 조건

- 주요 에러 케이스 처리
- 저장 실패 시 복구 경로 확보
- 테스트 범위 확대
- 모바일 사용성 보정

## Phase 3: 핵심 기능 확장

### 목표

학습 흐름이 이어지도록 복습센터, 연습 모드, 기본 통계, 결과 화면 보강, 문제 유형 확장을 제공한다.

### 포함 기능

- 연습 모드
- 복습센터
- `Daily_Stats` 표시
- 결과 화면 강화
- 기본 통계 화면
- 문제 유형 추가
- 실연동 준비 문서와 GAS 스켈레톤
- mock/real 모드 분기와 연결 상태 표시

### 제외 기능

- 업적 완성형
- 이미지 문제
- 듣기 모드 고도화
- 자기채점 작문 완성형
- 다국어 추가
- 관리자 UI

### 현재 상태

- 복습센터, 통계, 연습 모드, 결과 화면 보강 완료
- `word_to_meaning`, `meaning_to_word` 2종 문제 유형 반영 완료
- mock/real runtimeConfig, apiClient, GAS 스켈레톤, 연결 문서 작성 완료
- 남은 핵심 작업은 문서 인코딩 정리와 실제 GAS/Google Sheets 구조 확정

### 완료 조건

- 연습/복습 흐름이 안정적으로 동작
- 기본 통계 화면 제공
- 문제 유형 2종 이상 안정화
- 실연동 준비 문서 정리
- 작업 기준 문서 최신화

### 다음 Phase 진입 전 확인 사항

- 복습 상태와 출제 우선순위가 맞는가
- 통계 데이터가 실제 저장 구조와 맞는가
- 모바일 UX가 충분히 안정적인가
